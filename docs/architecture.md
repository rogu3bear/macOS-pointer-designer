# Architecture

## Overview
WindowDrop's website is a static Single Page Application (SPA) built with Rust and WebAssembly.

## Stack
- **Framework**: [Leptos](https://github.com/leptos-rs/leptos) (v0.6) in CSR (Client-Side Rendering) mode.
- **Build Tool**: [Trunk](https://trunkrs.dev/).
- **Routing**: `leptos_router`.
- **Styling**: Hand-written CSS using CSS variables (`styles/`).
- **Markdown**: `pulldown-cmark` for rendering content at runtime (compiled via `include_str!`).

## Directory Structure
- `site/src/`: Application source.
- `site/styles/`: Global and component CSS.
- `site/assets/`: Static SVGs.
- `site/public/`: Root-level files (robots.txt, sitemap.xml).
- `dist/`: The output directory generated by `trunk build`.

## Deployment
The `dist/` folder is self-contained and can be deployed to any static host (Cloudflare Pages, Vercel, Netlify, GitHub Pages).
Ensure the host is configured to rewrite 404s to `index.html` for client-side routing to work.

## CTA System

The site uses a config-driven CTA (Call-to-Action) system to support different monetization phases without code changes.

### Configuration (`src/config.rs`)

```rust
pub enum CtaMode {
    TrialDownload,  // Direct download link
    EarlyAccess,    // Same as trial, different label
    NotifyMe,       // Email capture form (pre-launch)
    AppStoreLink,   // Mac App Store link
}
```

The active mode determines:
- Primary CTA button label
- Button behavior (link vs. email capture)
- Whether email capture section is shown

### Components

- `CtaPrimary`: Renders the primary CTA based on config mode
- `CtaSecondary`: Static secondary button (e.g., "How it works")
- `EmailCapture`: Email signup form for pre-launch phase

## Analytics

Minimal interest tracking via console logging (MVP approach).

### Events Tracked

| Event | Trigger |
|-------|---------|
| `cta_primary_clicked` | Primary CTA button clicked |
| `cta_secondary_clicked` | Secondary CTA button clicked |
| `animation_viewed` | Hero animation completed |
| `waitlist_submitted` | Email form submitted |

### Implementation (`src/analytics.rs`)

In development builds, events are logged to the browser console for debugging. Production builds do not emit analytics/telemetry. To integrate with an analytics provider, update the `track_event` function.

## Hero Animation

SVG-based animation demonstrating WindowDrop behavior.

### Storyboard
1. Cursor moves into position
2. Keys overlay: "⌘ N"
3. Window appears under cursor
4. Settle animation
5. Caption appears

### Accessibility
- Respects `prefers-reduced-motion` media query
- Static fallback shows final state
- Proper `aria-label` for screen readers
